## 💡 核心功能展示

### Node 1: 智能规划提取
```
输入: "我有5个土豆、250g五花肉、3个番茄，
       4个人吃饭，要做2菜1汤，不吃辣"

输出: UserMenuConstraints {
  available_ingredients: [
    {"name": "土豆", "quantity": "5个"},
    {"name": "五花肉", "quantity": "250g"},
    {"name": "番茄", "quantity": "3个"}
  ],
  allergies_and_dislikes: ["辣"],
  portion_size: 4,
  global_requests: "2菜1汤",
  search_queries: [6-8 个搜索词]
}
```

### Node 2: 多源并发检索
```
6-8 个搜索词
    ↓
并发执行 (最多 3 个并发)
    ├─ 策略 1: MCP (小红书)      → 失败/成功
    ├─ 策略 2: Schema (下厨房)   → 失败/成功
    ├─ 策略 3: Bing (搜索)       → 失败/成功
    └─ 策略 4: Playwright (爬取) → 失败/成功/熔断

结果: 4-5 条参考菜谱
```

### Node 3: 智能菜谱生成
```
输入: 食材库存 + 参考菜谱 + 忌口约束

处理:
  1. 过滤与忌口冲突的菜谱
  2. 精准分配食材（零超发）
  3. 向用户说明分配逻辑
  4. 生成完整菜谱

输出: Markdown 格式菜谱 (流式推送)
  ### 👨‍🍳 总厨算账总结
  ### 番茄土豆汤
  ### 五花肉炒豆角
  ...
```

## 🌟 项目亮点

### ✨ 三个核心创新

#### 1. 库存精准分配
- **问题**: 一般菜谱不考虑用户的实际库存
- **解决**: 
  - ✅ 精确量化每个食材的数量单位
  - ✅ 计算所有菜品消耗总和
  - ✅ Self-Audit 机制向用户说明分配
  - ✅ 绝对零库存超发

#### 2. 四层检索瀑布流
- **问题**: 单一数据源易失败，爬虫易被封
- **解决**:
  - ✅ MCP (官方接口) → Schema 提取 → Bing 搜索 → Playwright
  - ✅ 单个策略失败自动切换，无缝降级
  - ✅ 验证码自动熔断，防止死循环
  - ✅ 多源数据融合

#### 3. 流式 Web 交互
- **问题**: 用户等待时间长，无进度反馈
- **解决**:
  - ✅ SSE 实时推送规划、检索、生成进度
  - ✅ 三个折叠面板动态展示各阶段
  - ✅ 打字机流式显示菜谱
  - ✅ 消除长等待阻塞
