# 🚀 快速开始指南

## 第一次设置（5 分钟）

### Step 1: 配置 API Keys

编辑 `.env` 文件（从 `.env.example` 复制）：

```env
# 必需：OpenAI API Key
OPENAI_API_KEY=sk-xxx...

# 推荐：Bing Search API Key
BING_SEARCH_API_KEY=xxx...

# 可选：小红书 MCP 配置（不配置也能用，会自动降级）
XHS_COOKIE=xxx...
A1=xxx...
```

**获取 API Keys：**
- OpenAI: https://platform.openai.com/api-keys
- Bing Search: https://www.microsoft.com/en-us/bing/apis/bing-web-search-api

### Step 2: 安装依赖

```bash
# 创建虚拟环境（推荐）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### Step 3: 启动服务

**方法 A: 一键启动脚本（推荐）**

- Windows: 双击 `start.bat`
- Linux/Mac: `bash start.sh`

**方法 B: 手动启动两个终端**

Terminal 1 - 启动后端：
```bash
python main.py
```
> 访问 http://localhost:8000/docs 查看 API 文档

Terminal 2 - 启动前端：
```bash
streamlit run app.py
```
> 自动打开 http://localhost:8501

## 常见场景示例

### 📍 场景 1: 晚餐排菜

在前端输入框输入：
```
我家里有：5个中等土豆、250克五花肉、3个番茄、半斤豆角、1棵小白菜。
今晚要招待一家人4个人吃饭，想做2菜1汤。
我和爸爸不能吃辣，妈妈喜欢清淡。
请帮我规划一下怎么做最好吃～
```

系统将：
1. ✅ 解析出食材数量与忌口
2. 🔍 从下厨房、Bing、小红书并发检索菜谱
3. 👨‍🍳 生成完美分配的 2 菜 1 汤方案

### 📍 场景 2: 快手菜

```
只有：2个番茄、1根黄瓜、2个鸡蛋
需要：快速做1菜，要能下饭的，5分钟搞定
```

### 📍 场景 3: 减脂餐

```
有：鸡胸肉200g、西兰花1棵、番茄2个、玉米1根
要求：减脂餐、少油、高蛋白、做1菜2配菜
```

## 🎯 核心工作流

```
用户输入
   ↓
[Node 1] LLM 规划提取 (提取食材、忌口、搜索词)
   ↓ (yield: planning_done)
   ├─ 显示提取的食材清单
   └─ 生成的搜索词
   ↓
[Node 2] 并发多策略检索 (MCP → Schema → Bing → Playwright)
   ↓ (yield: retrieval_update)
   ├─ 土豆番茄清淡做法 ✅
   ├─ 五花肉豆角咸香 ✅
   ├─ 不辣汤 ✅
   └─ ...
   ↓
[Node 3] LLM 生成菜单 (融合参考数据，精准库存分配)
   ↓ (yield: recipe_stream)
   └─ 实时流式显示菜谱
```

## 📊 输出示例

系统最终生成的菜单格式：

```markdown
### 👨‍🍳 总厨算账总结

为了满足您4人、2菜1汤的需求，我做如下分配：
- 3个番茄 + 5个土豆 → 番茄土豆汤
- 250g五花肉 + 豆角 → 五花肉炒豆角
- 白菜 + 番茄 → 清汤白菜汤（补充）

这样不仅完美避开了辣，还能兼顾妈妈的清淡需求。

### 番茄土豆汤

#### 精确用料
- 番茄：2个（去籽，切块）
- 土豆：3个（削皮，切块）
- 清水：800ml
- 盐：适量
- 糖：1小勺（中和酸味）

#### 烹饪步骤
1. 番茄冷水下锅，煮 3 分钟至开花，转大火
2. 放入土豆块，保持大火 2 分钟
3. 转中火继续煮 8-10 分钟直到土豆软烂
4. 最后加盐和糖调味，稍煮 1 分钟

### 五花肉炒豆角

#### 精确用料
- 五花肉：250g（切片）
- 豆角：200g（斜切）
- 大蒜：3 瓣（切片）
- 豆瓣酱：2 勺（无辣，改用咸豆豉）
- 油：1 勺

#### 烹饪步骤
1. 热锅下油，五花肉中火煸出油脂，约 3 分钟
2. 加豆角继续炒，让豆角受热均匀，约 4 分钟
3. 加豆豉和蒜片，炒香，1 分钟
4. 转大火，快速翻炒使豆角微微焦香，2-3 分钟即可出锅

### 清汤白菜汤

#### 精确用料
- 小白菜：1 棵（切段）
- 番茄：1 个（块状）
- 清水：600ml
- 盐：适量

#### 烹饪步骤
1. 冷水加番茄，烧开后转中火 3 分钟
2. 放入白菜段，煮 2-3 分钟至汤色变浅黄
3. 加盐调味，即可出锅
```

## 🔧 故障排除

### ❌ "无法连接到后端"

**原因：** 后端服务未启动或地址错误

**解决：**
```bash
# 检查后端是否启动
python main.py

# 检查地址（应在 http://localhost:8000）
# 更新前端 .env 中的 BACKEND_URL
```

### ❌ "OPENAI_API_KEY not found"

**原因：** 环境变量未正确配置

**解决：**
```bash
# 确保 .env 文件存在
cp .env.example .env

# 编辑 .env，填入实际 API Key
# Windows: 记事本 .env
# Linux: nano .env
```

### ❌ "MCP 策略未启用：缺少 XHS_COOKIE"

**这是正常的，** MCP 是可选的。系统会自动降级到其他策略（Schema、Bing、Playwright）。

### ❌ "请求超时"

**原因：** 网络不稳定或后端处理时间过长

**解决：**
- 检查网络连接
- 等待 30-60 秒（首次请求可能需要初始化）
- 减少搜索词数量

### ❌ "检测到验证码，熔断查询"

**这是正常的防反爬机制。** 系统会自动尝试其他策略。

**优化建议：**
在 `.env` 中增加延迟时间：
```env
RANDOM_DELAY_MIN=2.5
RANDOM_DELAY_MAX=5.0
```

## 📈 性能优化

### 提升规划质量

在 Node 1 提示词中，system prompt 会根据您的输入自动生成 4-8 个搜索词。

**提示：** 输入越详细，搜索词越精准，检索效果越好。

### 加快检索速度

```env
# 增加并发任务数（1-5，默认 3）
# 需要在 retrieval.py 中修改
max_concurrent = 5
```

### 减少反爬问题

```env
# 增加随机延迟（秒）
RANDOM_DELAY_MIN=2.0
RANDOM_DELAY_MAX=5.0
```

- 不要隐瞒重要忌口信息
